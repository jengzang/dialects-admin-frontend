# 单个用户会话管理界面优化说明

## 设计理念

对于单个用户的会话管理页面，应该聚焦于**会话本身的状态和行为**，而不是用户的账户信息。

## 原有设计问题

### 被注释掉的内容
1. **用户统计卡片**（5个）
   - 登录次数
   - 登录失败次数
   - 注册 IP
   - 总在线时长
   - 最近一次登录

   **问题：** 这些是用户账户级别的统计，不是会话级别的信息。

2. **近期登录 IP 统计**
   - **问题：** 这个其实与会话直接相关，不应该被注释掉。

## 优化方案

### 新增：会话概览统计卡片（6个）

```
┌─────────────────────────────────────────────────────────┐
│ [总会话数] [活跃会话] [可疑会话] [已撤销] [使用IP数] [使用设备数] │
└─────────────────────────────────────────────────────────┘
```

#### 1. 总会话数
- **含义：** 该用户的所有会话总数（包括活跃、过期、已撤销）
- **用途：** 了解用户的会话规模
- **颜色：** 主题色（绿色）

#### 2. 活跃会话
- **含义：** 当前有效且未过期的会话数
- **用途：** 快速识别用户当前在线状态
- **颜色：** 成功色（绿色 #67c23a）
- **计算：** `!revoked && expires_at > now`

#### 3. 可疑会话
- **含义：** 被标记为可疑的会话数
- **用途：** 安全监控，快速发现异常
- **颜色：** 危险色（红色 #f56c6c）
- **计算：** `is_suspicious === true`

#### 4. 已撤销
- **含义：** 已被管理员撤销的会话数
- **用途：** 了解管理员干预历史
- **颜色：** 信息色（灰色 #909399）
- **计算：** `revoked === true`

#### 5. 使用 IP 数
- **含义：** 该用户使用过的不同 IP 地址数量
- **用途：** 识别异常登录地点（IP 数过多可能异常）
- **颜色：** 主题色
- **计算：** `unique(current_ip)`

#### 6. 使用设备数
- **含义：** 该用户使用过的不同设备数量
- **用途：** 识别设备共享或账号共享行为
- **颜色：** 主题色
- **计算：** `unique(device_fingerprint)` 或 fallback 到 `device_info`

### 保留：近期登录 IP 统计

**为什么保留：**
- 与会话直接相关（每个会话都有 IP）
- 帮助识别异常登录地点
- 提供地理分布信息

**显示内容：**
- IP 地址
- 使用次数
- 最后使用时间

## 实现细节

### 计算逻辑

```javascript
const sessionStats = computed(() => {
  const allSessions = sessions.value;
  const total = allSessions.length;
  const active = allSessions.filter(s =>
    !s.revoked && new Date(s.expires_at) > new Date()
  ).length;
  const suspicious = allSessions.filter(s => s.is_suspicious).length;
  const revoked = allSessions.filter(s => s.revoked).length;

  // 统计唯一IP数
  const uniqueIPs = new Set(allSessions.map(s => s.current_ip)).size;

  // 统计唯一设备数
  const uniqueDevices = new Set(
    allSessions
      .filter(s => s.device_fingerprint)
      .map(s => s.device_fingerprint)
  ).size;

  return {
    total,
    active,
    suspicious,
    revoked,
    unique_ips: uniqueIPs,
    unique_devices: uniqueDevices
  };
});
```

## 使用场景

### 场景 1：安全审计
管理员查看用户会话，快速识别异常：
- **可疑会话数 > 0** → 需要关注
- **使用 IP 数过多** → 可能账号共享或被盗
- **使用设备数过多** → 可能多人使用同一账号

### 场景 2：会话管理
管理员需要撤销用户会话：
- **活跃会话数** → 了解当前在线状态
- **总会话数** → 了解会话规模
- **已撤销数** → 了解历史干预情况

### 场景 3：行为分析
分析用户的登录行为：
- **近期登录 IP 统计** → 地理分布
- **使用设备数** → 设备偏好
- **会话列表** → 详细行为记录

## 数据来源

所有统计数据都是**实时计算**的，基于当前加载的会话列表：

```javascript
// 数据来源
sessions.value = await userSessionAPI.getUserHistory(userId, includeRevoked);

// 实时计算
const sessionStats = computed(() => {
  // 基于 sessions.value 计算统计数据
});
```

**优点：**
- 无需额外 API 请求
- 数据始终与会话列表同步
- 筛选条件（显示已撤销、仅显示可疑）会影响统计结果

## 与全局管理页面的对比

### 全局管理页面（GlobalSessionManagement）
- **统计范围：** 所有用户的所有会话
- **统计卡片：** 8个（总会话、活跃、撤销、过期、可疑、在线用户、总在线时长、平均时长）
- **用途：** 系统级监控

### 单用户管理页面（UserSessionManagement）
- **统计范围：** 单个用户的所有会话
- **统计卡片：** 6个（总会话、活跃、可疑、撤销、使用IP数、使用设备数）
- **用途：** 用户级监控

## 总结

### 设计原则
1. **聚焦会话** - 只显示与会话直接相关的统计
2. **实时计算** - 基于当前数据，无需额外请求
3. **安全导向** - 突出可疑会话和异常指标
4. **操作便捷** - 快速识别需要关注的问题

### 关键指标
- ✅ **总会话数** - 规模
- ✅ **活跃会话** - 在线状态
- ✅ **可疑会话** - 安全风险
- ✅ **已撤销** - 干预历史
- ✅ **使用 IP 数** - 地理分布
- ✅ **使用设备数** - 设备偏好

### 移除的指标（与会话无关）
- ❌ 登录次数 - 账户级统计
- ❌ 登录失败次数 - 账户级统计
- ❌ 注册 IP - 账户级信息
- ❌ 总在线时长 - 账户级统计
- ❌ 最近一次登录 - 账户级信息

这些信息更适合放在"用户详情"页面，而不是"会话管理"页面。
